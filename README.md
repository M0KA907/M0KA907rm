## Working style

```txt
1. Make it run.
2. Make it clear.
3. Make it fast enough.
4. Remove the junk.
5. Document the rollback.
```

I prefer small commits, plain files, reproducible commands, visible proof, and documentation that helps the next person fix the project without guessing.

---

## Useful repo standards

When I care about a repo, I try to leave it with:

- a short README that explains what the thing does
- setup and run commands
- screenshots or visible proof when possible
- known issues and rollback notes
- no mystery dependencies
- no fake complexity for aesthetic reasons
